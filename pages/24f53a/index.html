    # 验证Case

    市场操作 今天的指定的市场的查询： curl http://127.0.0.1:18080/ -d '{"method": "market.status_today", "params": ["BTCBCH"], "id": 1516681174}' 指定的市场的最新价格： curl http://127.0.0.1:18080/ -d '{"method": "market.last", "params":["BTCBCH"], "id":1}' 指定市场的指定时间区间的： curl http://127.0.0.1:18080/ -d '{"method": "market.status", "params": ["BTCBCH", 120], "id": 1}'

    支持的交易对（市场）： curl http://127.0.0.1:18080/ -d '{"method": "market.list", "params": [], "id": 1516681174}'

    指定的市场的成交历史： curl http://127.0.0.1:18080/ -d '{"method": "market.deals", "params":["BTCBCH",100, 10000], "id":1}' method: market.deals params: market: limit: count，no more than 10000 last_id: id limit

    指定用户的市场的成交历史： curl http://127.0.0.1:18080/ -d '{"method": "market.user_deals", "params":[10,"BTCBCH",0, 100], "id":1}'

    method: market.user_deals params: user_id: user ID，Integer market: market name，String offset: offset，Integer limit: limit，Integer

    获取k线: curl http://127.0.0.1:18080/ -d '{"method":"market.kline","params":["BTCBCH",1655037800,1655037840,60], "id":1}'

    method: market.kline params: market: market start: start，Integer end: end, Integer interval: interval, Integer

    交易操作 限价交易： curl http://127.0.0.1:18080/ -d '{"method": "order.put_limit", "params":[2, "BTCBCH", 2, "10", "8000", "0.002", "0.001", "hello"], "id":1111111 }'

    method: order.put_limit params: user_id: user ID，Integer market: market name，String side: 1: sell, 2: buy，Integer amount: count，String pride: price，String taker_fee_rate: String, taker fee maker_fee_rate: String, maker fee source: String, source，up to 30 bytes

    市价交易： curl http://127.0.0.1:18080/ -d '{"method": "order.put_market", "params":[20, "BTCBCH", 2, "10", "0.002", "hello"], "id":20 }' method: order.put_market params: user_id: user ID，Integer market: market name，String side: 1: sell, 2: buy，Integer amount: count or amount，String taker_fee_rate: taker fee source: String, source，up to 30 bytes

    取消订单: method: order.cancel params: user_id: user ID market：market order_id： order ID

    订单详情: method: order.deals params: order_id: order ID, Integer offset limit

    订单列表: curl http://127.0.0.1:18080/ -d '{"method": "order.book", "params":["BTCBCH", 2, 0, 100], "id":1}'

    method: order.book params: market: side: side，1：sell，2：buy offset:(0) limit:(最大101)

    订单深度： curl http://127.0.0.1:18080/ -d '{"method": "order.depth", "params":["BTCBCH", 101, "60"], "id":1}' method: order.depth params: market：market name limit: count limit，Integer interval: interval，String, e.g. "1" for 1 unit interval, "0" for no interval

    某user的pending订单列表： curl http://127.0.0.1:18080/ -d '{"method": "order.pending", "params":[3, "BTCBCH", 0, 100], "id":1}' method: order.pending params: user_id: user ID，Integer market: market name，String offset: offset，Integer limit: limit，Integer

    某个订单ID的详情（其实比上面pending没多信息）： curl http://127.0.0.1:18080/ -d '{"method": "order.pending_detail", "params":["BTCBCH", 1], "id":1}' method: order.pending_detail params: market: order_id: order ID，Integer

    资产操作： 查询资产 curl http://127.0.0.1:18080/ -d '{"method": "balance.query", "params":[1], "id":1 }'

    method: balance.query params: unfixed parameters, first parameter is user ID, followed by list of asset names. Return to user's overall asset if the list is blank.

    改变资产: curl http://127.0.0.1:18080/ -d '{"method": "balance.update", "params":[1,"BTC", "deposit", 100, "1.2345", ""a""], "id":1 }'

    method: balance.update params: user_id: user ID，Integer asset: asset name，String business: business type，String business_id: business ID(同一个id只会更新一次，用于幂等)，Integer, but it will only succeed once with multiple operations of the same user_id, asset, business or business_id change: balance change，String, negative numbers for deduction detail: Json object，attached information

    支持的资产： curl http://127.0.0.1:18080/ -d '{"method": "asset.list", "params":[], "id":0}'

    内存所有资产的总额： curl http://127.0.0.1:18080/ -d '{"method": "asset.summary", "params":[], "id":0}'

    查询一个user的账户变动历史： curl http://127.0.0.1:18080/ -d '{"method": "balance.history", "params":[1,"BTC", "", 0, 0, 0, 10], "id":1}'

    method: balance.history params: user_id: user ID, Integer asset: asset name，which can be null business: business，which can be null，use ',' to separate types start_time: start time，0 for unlimited，Integer end_time: end time，0 for unlimited, Integer offset: offset position，Integer limit: count limit，Integer

    # 代码说明

    # accesshttp

    init_methods_handler， 输入的method的所有参赛值都在这里初始化的 on_http_request, 对外的接口

    失败没有回滚的操作，比如order_put里面的balance操作失败，都没有回滚操作 多开了几个线程，把成交的记录异步刷到mysql

    # network

    标准的reactor模式，底层是libev这个开源的简单的异步框架；

    nw_sock封装了socker相关的函数(对setsockopt，getpeername, 地址格式, fcntl)；

    nw_evt简单的封装了ev_loop

    nw_timer对计时器的简单封装，主要是对一些不停复用的参数、操作封装到了函数内部

    nw_buf对buffer的管理封装(buf是带有自己管理自己的，用buf_pool和buf是一套；cache是不会自己管理自己的，需要自己去组织这些cache，相当于它不带头，只相当于一个内存池，每个大小的data对应一个独立的cache pool，buf-pool也一样，其实就是类似c++的模板，假如c++实现，其实就是这里void变成了类型，然后输入参数size去掉，变成编译器自己通过类型参数去计算这个size）

    nw_buf是一个buf的实例，多个buf用链表连接，记录了当前data的大小，和data的指针  

nw_buf_pool是一个buf的池，nw_buf的分配和回收都从这个池子中处理，它负责真正的分配和回收，小于65536个buf，不会回收给os；  

nw_buf_list是一个buf的链表的表头  

nw_cache是一个和pool类似的结构体，，内部字段唯一区别是free_arr类型不一样

    nw_ses对socket的，evloop的事件的简单封装，可以读写一个ses，把时间隐藏起来，关联到读写网络的回调上，读写用到了上面的buf pool

    nw_svr对server的socket、cli的socket的交互的封装，可以listen多个addr；可以不同addr

    nw_clt:是对client的封装，可以断线自动重连，对res的一些不涉及client方面的接口做了屏蔽，公共的实现放在这个封装里面：on_connect, on_error, on_close

    nw_job其实就是初始化的时候启动几个线程（nw_job），然后不停的从队列里面去获取任务去执行任务(nw_job_entry)，任务怎么执行是规定的回调接口

    （nw_job_type），然后使用方只需要把任务丢到队列里面，它就可以异步执行完成；然后通过pipe管道做异步的通知到主线程，主线程监听这个pip的read事件；

    疑问点：add-job，和pip的read事件触发之后，都是是要通过获取锁，去访问公共数据结构链表的，这里可能灰度导致主线程睡眠，主线程最好做的不睡眠，无锁的通知；

    优化点：add-job，其实是一写多读，可以优化成无锁的单向链表；pipe也一样，它是多写一读，也可以做成无锁单向链表，还是通过pipe做通知，通知需要告诉主线程是哪个队列（cas，ref复制）；

    或者add和reply回调的信息，都通过pipe完整的传输，而不是仅仅通过pipe做通知，最终还是得去竞争锁；

    nw_state: 其实就是一个带超时的缓存，用哈希表存信息，超时则回调；比如一个request发了之后，通过这个state缓存这个req的seq，超时则处理，没超时就正常返回则从stat删除这个seq的信息；
